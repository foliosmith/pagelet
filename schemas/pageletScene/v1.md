# pageletScene Binary Schema v1

Status: active. This schema version is independent from the `pagelet` crate
semantic version.

All multi-byte numbers use little-endian byte order. Encoders must produce
fields in the order below. Decoders must reject unknown schema versions,
payload kinds, enum discriminants, invalid booleans, invalid ranges, and
trailing bytes.

## Envelope

| Offset | Size | Field | Value |
|---:|---:|---|---|
| 0 | 8 | magic | ASCII `PGLTSCN` followed by `00` |
| 8 | 2 | schema version | `1` |
| 10 | 2 | payload kind | `1` PageBatch; `2` MeasureBatch; `3` MeasuredBatch |
| 12 | 4 | payload length | exact byte length after the header |
| 16 | 4 | payload checksum | CRC-32/ISO-HDLC of payload bytes |

The fixed header is 20 bytes. The maximum payload is 64 MiB.

## Primitive encodings

- `u8`, `u16`, `u32`, `u64`, and `i64` use their fixed-width representation.
- `bool` is one byte: `0` false, `1` true. Other values are invalid.
- `LayoutUnit` is its raw `i64` fixed-point value, where 64 units equal one
  logical pixel.
- `string` is `u32 byte_length` followed by valid UTF-8 bytes.
- `vec<T>` is `u32 item_count` followed by the items.
- `option<T>` is a `bool` followed by `T` only when present.
- `ContentHash` is its raw 32 bytes.
- Collections are limited to 1,000,000 items and strings to 8 MiB.

## PageBatch (kind 1)

```text
vec<PageScene> pages
```

### PageScene

Fields are encoded in this exact order:

```text
u32 page_index
PageSize size
option<TextAnchor> start_anchor
option<TextAnchor> end_anchor
vec<SceneFragment> fragments
vec<LinkRegion> links
vec<AnchorRegion> anchors
vec<SelectionMap> selections
vec<SemanticNode> semantics
ContentHash fingerprint
option<BreakToken> next_break_token
vec<Diagnostic> diagnostics
```

Supporting records:

```text
PageSize       = LayoutUnit width, LayoutUnit height
Rect           = LayoutUnit x, LayoutUnit y, LayoutUnit width, LayoutUnit height
SourceRange    = u32 start, u32 end
TextAnchor     = u32 document_id, u32 node_id, u32 utf8_byte_offset,
                 u8 affinity
TextAnchorRange = TextAnchor start, TextAnchor end

SceneFragment  = u32 id, u8 kind, u32 node_id, Rect rect,
                 option<string> text, option<SourceRange> source_range,
                 option<TextAnchorRange> anchor_range,
                 option<u32> line_index, bool overflow

LinkRegion     = Rect rect, u32 node_id, string href,
                 option<string> resolved_document,
                 option<string> fragment, u8 kind

AnchorRegion   = Rect rect, string key, u32 node_id

SelectionMap   = u32 node_id, u32 start, u32 end, vec<Rect> rects

SemanticNode   = u32 node_id, Rect rect, string role, string label

BreakToken     = u32 node_id, u32 child_index, u32 text_offset,
                 bool continuation, u32 page_index,
                 ContentHash content_fingerprint,
                 u64 config_fingerprint, u64 text_backend_id,
                 u64 font_fingerprint

Diagnostic     = u8 code, u8 severity, string message,
                 option<u32> resource_id,
                 option<SourceRange> source_range
```

Page enum discriminants:

| Type | Values |
|---|---|
| TextAffinity | `0` Upstream, `1` Downstream |
| SceneFragmentKind | `0` TextLine, `1` Marker, `2` Image, `3` Divider, `4` BackgroundBorder, `5` DebugOverlay, `6` UnsupportedPlaceholder |
| LinkKind | `0` Internal, `1` External, `2` Resource, `3` Footnote, `4` Unknown |
| Severity | `0` Info, `1` Warning, `2` Error |
| DiagnosticCode | `0` Io, `1` InvalidContainer, `2` InvalidPackage, `3` UnsupportedFeature, `4` ResourceLimitExceeded, `5` Parse, `6` Layout, `7` Cancelled, `8` Protocol, `9` Internal |

## MeasureBatch (kind 2)

```text
vec<MeasureRequest> requests
```

### MeasureRequest

```text
u32 id
u32 paragraph_id
string text
u32 text_range_start
u32 text_range_end
vec<TextStyleRun> style_runs
LayoutUnit font_size
LayoutUnit max_width
LayoutUnit available_width
string locale
u8 direction
LayoutUnit text_scale
FontFallbackChain font_candidates
StrutStyle strut
u8 height_behavior
u64 request_fingerprint
```

Supporting records:

```text
TextStyleRun       = u32 start, u32 end, LayoutUnit font_size,
                     FontFallbackChain fonts
FontFallbackChain  = FontDescriptor primary,
                     vec<FontDescriptor> fallbacks
FontDescriptor     = option<u32> font_id, string family, u16 weight,
                     u8 style, u16 stretch, u64 fingerprint
StrutStyle         = LayoutUnit ascent, LayoutUnit descent,
                     LayoutUnit leading
```

Measurement enum discriminants:

| Type | Values |
|---|---|
| TextDirection | `0` Auto, `1` Ltr, `2` Rtl |
| FontStyle | `0` Normal, `1` Italic, `2` Oblique |
| HeightBehavior | `0` Natural, `1` IncludeStrut, `2` Tight |

## MeasuredBatch (kind 3)

```text
u64 text_backend_id
u64 font_fingerprint
vec<MeasuredText> results
```

```text
MeasuredText = u32 request_id, u64 request_fingerprint,
               LayoutUnit width, LayoutUnit height,
               u32 line_count, u32 utf8_len,
               vec<LineMetrics> lines, vec<TextCluster> clusters,
               u64 measurement_fingerprint
LineMetrics  = u32 text_start, u32 text_end,
               LayoutUnit baseline, LayoutUnit ascent, LayoutUnit descent,
               LayoutUnit line_height, LayoutUnit width, bool hard_break
TextCluster  = u32 text_start, u32 text_end, u32 line_index,
               LayoutUnit x_start, LayoutUnit x_end
```

The response echoes both the request id and request fingerprint. Layout rejects
missing, duplicate, unknown, stale, invalid UTF-8, or out-of-order results before
using host metrics.

`text_range` and every `TextStyleRun` range use UTF-8 byte offsets. They must
be ordered, remain inside `text`, land on UTF-8 character boundaries, and each
style run must remain inside `text_range`.

## Compatibility policy

- v1 records are positional and have no unknown-field extension area. Adding,
  removing, reordering, or changing any field or record enum discriminant
  requires a new schema version. A new top-level payload kind is additive: old
  decoders reject the unknown kind without misreading an existing payload.
- A decoder for v1 rejects payloads from all other versions rather than
  guessing compatibility.
- Canonical encoding is the exact field order and primitive representation in
  this document. Decoding and re-encoding a valid payload must reproduce the
  same bytes.
- Crate releases that do not change this document keep schema version `1`,
  regardless of the crate semver.
