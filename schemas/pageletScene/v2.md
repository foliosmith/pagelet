# pageletScene Binary Schema v2

Status: active. This version adds exact paragraph replay to the frozen v1
schema. It is independent from the `pagelet` crate semantic version.

The 20-byte envelope, primitive encodings, allocation limits, common page
records, enum discriminants, and payload kinds are unchanged from
[`v1`](v1.md), except that the envelope schema version is `2`.

## PageBatch (kind 1)

```text
vec<PageScene> pages
```

`PageScene` fields are positional and encoded in this exact order:

```text
u32 page_index
PageSize size
option<TextAnchor> start_anchor
option<TextAnchor> end_anchor
u64 text_backend_id
u64 font_fingerprint
vec<SceneParagraph> paragraphs
vec<TextPaintFragment> text_paints
vec<SceneFragment> fragments
vec<LinkRegion> links
vec<AnchorRegion> anchors
vec<SelectionMap> selections
vec<SemanticNode> semantics
ContentHash fingerprint
option<BreakToken> next_break_token
vec<Diagnostic> diagnostics
```

`SceneFragment` is retained for markers, images, rules, backgrounds, debug
geometry, unsupported placeholders, and decoded legacy content. New text is
represented only by the paragraph table and paint references.

```text
Point = LayoutUnit x, LayoutUnit y

SceneParagraph =
  u32 paragraph_id
  u64 request_fingerprint
  u64 measurement_fingerprint
  string text
  u32 text_range_start
  u32 text_range_end
  vec<TextStyleRunV2> style_runs
  LayoutUnit font_size
  LayoutUnit available_width
  LayoutUnit max_width
  string locale
  u8 direction
  LayoutUnit text_scale
  FontFallbackChain font_candidates
  StrutStyle strut
  u8 height_behavior
  vec<SceneParagraphLine> lines
  vec<TextCluster> clusters

SceneParagraphLine =
  LineMetricsV2 metrics
  Rect layout_rect
  Rect ink_bounds

TextPaintFragment =
  u32 id
  u32 node_id
  u32 paragraph_id
  u32 visible_text_start
  u32 visible_text_end
  Point paint_origin
  Rect layout_rect
  Rect clip_rect
  u32 first_line
  u32 line_count
  option<SourceRange> source_range
  TextAnchorRange anchor_range
  bool overflow
```

`paint_origin` is the page position of the complete paragraph's local origin.
A paragraph split over multiple pages retains the same paragraph ID and both
fingerprints. Each page changes only its visible range, first line/count,
origin, and page clip. `clip_rect` is a page-fragment clip and must not be
derived from line advance width. Selection geometry remains in
`SelectionMap.rects`.

## MeasureBatch (kind 2)

The v1 `MeasureRequest` field order is retained. `TextStyleRun` changes to:

```text
TextStyleRunV2 = u32 start, u32 end, LayoutUnit font_size,
                 LayoutUnit letter_spacing, FontFallbackChain fonts
```

Font descriptors carry family, weight, style, stretch, and font fingerprint.
Together with locale, direction, scale, strut, ranges, and wrapping widths,
these values participate in the request fingerprint.

## MeasuredBatch (kind 3)

The v1 `MeasuredBatch` and `MeasuredText` field order is retained.
`LineMetrics` changes to:

```text
TextBounds = LayoutUnit x, LayoutUnit y,
             LayoutUnit width, LayoutUnit height

LineMetricsV2 = u32 text_start, u32 text_end,
                LayoutUnit baseline, LayoutUnit ascent, LayoutUnit descent,
                LayoutUnit line_height, LayoutUnit width,
                TextBounds ink_bounds, bool hard_break
```

The baseline and ink bounds are authoritative host results. Layout does not
reconstruct the baseline from line height. Ink may extend outside advance width
and therefore must not be clipped to `width`.

## Compatibility and cache policy

- The current decoder dispatches v1 and v2 by envelope version. It never tries
  to parse one positional schema as the other.
- Decoded v1 style runs default `letter_spacing` to zero. Decoded v1 line ink
  defaults to `(0, 0, width, line_height)`. Decoded v1 pages have empty
  paragraph/paint tables and zero backend/font identity.
- Explicit v1 encoding projects v2 text paints to legacy per-line
  `SceneFragment::TextLine` records. This is a capability fallback, not the v2
  replay path.
- v2 bumps `EngineVersions.style_schema`, `text_schema`,
  `pagination_algorithm`, `scene_wire`, and `disk_cache`. Old derived caches
  are invalidated.
- Reuse requires exact backend ID, font fingerprint, request fingerprint, and
  measurement fingerprint matches for every referenced paragraph.
